### ShelfLifeRule（FoodKeeper データ）の課題 — 2026-06-10 調査結果

公式 USDA 配布元（fsis.usda.gov の EN/ES/PT JSON・XLS、CC0 ライセンス）が**この開発環境から Akamai により 403 でブロック**され、自動取得できない（curl の UA 変更・WebFetch とも不可。data.gov のリソース URL は 404）。

- 代替として GitHub ミラー `jelera/food-shelflife-db` の `lib/seeds/ingredients.csv`（663行、FoodKeeper スキーマ: Pantry/Refrigerate/Freeze の Min/Max/Metric + 開封後・解凍後）が raw.githubusercontent.com から取得可能なことを確認済み。
- 未決定事項（ユーザー判断が必要）:
  1. **取得経路**: ユーザーがブラウザで公式 XLS/JSON を手動ダウンロードして `doc/` 等に配置するか、ミラー CSV を使うか（ミラーはデータの鮮度・正確性が公式と一致する保証なし）
  2. **日本語マッピング方式**: FoodKeeper の英語食材名 → 日本語名・`normalizedName` への対応付けをどう作るか（AI 一括翻訳 + 人手レビューを想定）
  3. **AI 和食材補完の実行方法**: ビルド時生成スクリプトをどう動かすか（API キーの扱い・実行タイミング。リポジトリにはスクリプトと生成済み JSON のみコミットする想定）
  4. **出典表記**: CC0 のため法的義務はないが、ヘルプ画面に USDA FoodKeeper 出典を記載する方針（ロードマップ #9 と連動）
