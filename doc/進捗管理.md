### ShelfLifeRule（FoodKeeper データ）の課題 — 2026-06-10 調査結果 → ✅ 2026-06-11 実装完了

> **2026-06-11 完了**: 下記の決定事項どおり、ミラー CSV（2019-06・CC0）+ 開発時マッピング（FoodKeeper 97 + 和食材補完 94）で実装。`tool/shelf_life/generate.dart` が `assets/shelf_life/shelf_life_rules.json`（185 ルール）を生成・同梱し、実行時は `ShelfLifeTable` が参照する。公式最新版に差し替えたくなったら `tool/shelf_life/README.md` の手順（ユーザーがブラウザで公式 JSON/XLS を入手 → 同ディレクトリに置く → 再生成）に従う。詳細は §2 の完了エントリ参照。

公式 USDA 配布元（fsis.usda.gov の EN/ES/PT JSON・XLS、CC0 ライセンス）が**この開発環境から Akamai により 403 でブロック**され、自動取得できない（curl の UA 変更・WebFetch とも不可。data.gov のリソース URL は 404）。

- 代替として GitHub ミラー `jelera/food-shelflife-db` の `lib/seeds/ingredients.csv`（663行、FoodKeeper スキーマ: Pantry/Refrigerate/Freeze の Min/Max/Metric + 開封後・解凍後。データは 2019-06 時点の FoodKeeper 由来）が raw.githubusercontent.com から取得可能なことを確認済み。
- **決定事項（2026-06-11 ユーザー判断）**:
  1. **取得経路**: ✅ ミラー CSV で実装を進め、公式版をユーザーがブラウザで入手できたら入力を差し替えて再生成する（差し替え前提のパイプライン構造にする）。FoodKeeper はデータ更新頻度が低く、本アプリの用途は「目安」のため 2019 年版で実用上問題なしと判断
  2. **日本語マッピング方式**: ✅ 開発時に Claude が対訳（日本語名・normalizedName → FoodKeeper 項目）を作成してコミットし、人手レビュー可能にする
  3. **AI 和食材補完の実行方法**: ✅ ビルド時に API は叩かない。FoodKeeper にない和食材（豆腐・納豆・味噌等）は開発時に Claude が日数データを作成してコミット
  4. **出典表記**: ✅ ヘルプ画面に USDA FoodKeeper 出典 + 「あくまで目安」を実装済み（M8）
