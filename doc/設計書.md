## 1. アーキテクチャ概要

- **feature-first** のフォルダ構成。各 feature 内を `domain` / `data` / `presentation` の3層に分ける
- 状態管理は **Riverpod**。UI は provider 経由で repository を呼ぶ
- ローカル永続化は **Drift**（SQLite）。repository が Drift をカプセル化し、UI から DB 詳細を隠す
- OS 依存機能（買い物リスト・同期）は**抽象インターフェース**を1つ切り、OS ごとに実装を差し替える
- レイアウトは OS ではなく**画面幅**（`LayoutBuilder`/`MediaQuery`）で分岐

```
UI (presentation)  ──watch/read──▶  Riverpod Provider  ──▶  Repository  ──▶  Drift (AppDatabase)
                                          │
                              抽象IF (RecipeProvider / ShoppingListService / SyncService) ※将来
```

## 4. データモデル

### 4.1 Drift スキーマ（✅ 実装済み）

**Ingredients**（生成データクラス名 `Ingredient`）
| 列 | 型 | 備考 |
|---|---|---|
| id | Text (PK) | UUID |
| name | Text | 表示名 |
| normalizedName | Text | 名寄せキー。AI連携前は name 流用 |
| category | Text(enum) | `textEnum<IngredientCategory>()`（列挙子名で保存）|
| quantity | Real | 小数可 |
| unit | Text | 定義済み=列挙子名 / カスタム=生文字列 |
| expiryDate | DateTime? | 任意 |
| updatedAt | DateTime | |

**SettingsTable**（生成データクラス名 `AppSettings`）— 単一レコード（id=0 固定）
| 列 | 型 | 備考 |
|---|---|---|
| id | Int (PK) | 既定 0 |
| localePref | Text | 'ja' / 'en' / 'system'（既定 'system'）|

- `schemaVersion = 1`。マイグレーションは将来追加
- テスト時は `AppDatabase(NativeDatabase.memory())` でインメモリ注入

## 6. 状態管理（Riverpod、✅ 現状）

| Provider | 種類 | 役割 |
|---|---|---|
| `databaseProvider` | Provider | AppDatabase を単一共有（onDispose で close）|
| `inventoryRepositoryProvider` | Provider | InventoryRepository を DI |
| `settingsRepositoryProvider` | Provider | SettingsRepository を DI |
| `categoryFilterProvider` | NotifierProvider | カテゴリフィルタ（null=すべて）|
| `inventoryListProvider` | StreamProvider | フィルタ追従の在庫一覧（ソート済み）|
| `selectedIngredientIdProvider` | NotifierProvider | master-detail の選択 id |
| `localeControllerProvider` | NotifierProvider | UI言語（DBと同期）|

- Riverpod 3 で `StateProvider` が legacy 化したため、可変状態は `Notifier` サブクラス（`set()` メソッド）で実装

## 7. 主要ロジック

### 7.1 在庫ソート（InventoryRepository）
SQLite の `ORDER BY expiryDate IS NULL, expiryDate ASC, updatedAt DESC`
→ 期限なしを末尾、期限近い順、同順は更新新しい順。

### 7.2 数量の増減（increment / decrement）
`max(0, quantity ± by)` に更新し updatedAt 更新。**0 でも行は削除しない**。詳細画面のステッパーと一覧操作で使用。
「使い切った」「削除」は明示削除（`deleteById`）で、UI はスナックバーの「元に戻す」で取り消し可能。

### 7.3 期限警告（expiry_status.dart）
カレンダー日で残り日数を算出 → `<0:expired / 0:today / 1〜3:warning / 4+:normal / null:none`。色は ColorScheme に対応付け。

### 7.4 日持ち初期値（shelf_life.dart）
`defaultExpiryFrom(category, from)` がカテゴリ目安日数を加算。目安なし（other）は null。参照箇所をこの関数に集約し、将来 JSON 同梱データへ差し替えやすくする。

### 7.5 単位表示（unit_option.dart）
`unitLabel(token, l10n)`: token が `UnitOption` の列挙子名ならローカライズ、なければカスタムとして生表示。保存値は言語非依存。

## 8. i18n 設計

- `l10n.yaml`: arb-dir=`lib/l10n`、template=`app_en.arb`、output=`app_localizations.dart`、`nullable-getter: false`
- カテゴリ/単位は enum 側に `label(AppLocalizations)` を持たせ一元化
- `MaterialApp.locale` に `localeControllerProvider` の値（null=端末追従）を渡す
- JSON（AIレスポンス）のフィールド名は言語固定、値のみ locale 反映（🔜）

## 9. 規約・方針

- **主キーは UUID(String)**（同期競合回避）
- **列挙子名・normalizedName は言語非依存キー**として保存（表示言語が変わっても不変）
- AI 出力は JSON 固定でパース、確認画面必須（カメラ）
- 日持ちデータは実行時に API を呼ばずビルド時同梱
- レイアウトは画面幅で分岐（`_tabletBreakpoint = 720`：狭い=単一ペイン / 広い=二ペイン master-detail）
